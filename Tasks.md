# Tasks

## Known Limitations (future work)

- [x] **Output filename collision on multi-part extensions** — Fixed in branch `feature/output-filename-collision`. Changed `src.with_suffix(fmt)` to `src.parent / (src.name + fmt)` in `app.py` so the new extension is always appended (`BUILDNGS.A-C` → `BUILDNGS.A-C.docx`) rather than replacing the existing one. 67/67 tests passing.

- [x] **Variable control prefix (`22 6d` vs `22 61`)** — Fixed in branch `feature/variable-ctrl-prefix`. Added `_detect_ctrl_byte()` which scans for the first `22 XX 0b` occurrence and returns `XX` (defaults to `0x61`). Threaded the detected byte through `_find_content_start`, `_skip_ctrl_sequence`, and the main parse loop. Files using `22 6d 0b` (e.g. `BUILDNGS.A-C`, `MARKETPL.*`) now produce correct content. 6 new tests. 73/73 tests passing.

- [x] **Running header/footer extraction** — Implemented in branch `feature/header-footer-extraction`. Added `_find_body_start` (c4 0e method + B5≠`0x14` fallback scan) and `_find_footer_start` (scans for section break between `first_para` and `body_start`). Position-based section routing at `22 XX 0b` blocks replaces the unreliable `_section_type_at` calls inside the parse loop. Added pre-body `00 00` NUL flush, B3=`0x00` layout-block skip, and `01 XX XX` 3-byte trailing indent fallback. Also fixed variable ctrl sequence skip (`i += 3` instead of `i += 4`) to prevent `0x13` formatting-prefix bytes being consumed before the main loop can recognise them. Footer page-number zone (centre-aligned `==` placeholder + page token machinery) is silently discarded; only the right-aligned document reference text is extracted. Confirmed on HENCOTES (header + footer) and BUILDNGS.H (footer only). 13 new tests. 89/89 tests passing.

- [x] **Tab stop position preservation** — Implemented in branch `feature/tab-stop-preservation`. Investigation documented in [`Tab Stop Investigation.md`](Tab%20Stop%20Investigation.md). Key changes: (1) `_detect_ctrl_byte()` now uses most-frequent `22 XX 0b` occurrence (not first) — fixes `Memorial.002` where header zone used `22 61 0b` but body used `22 42 0b`; (2) `Paragraph` gains `tab_stops: list[int]` field storing explicit stop positions in twips; (3) `parse()` reads scale pitch from layout table at `0x2C6+11` and computes `twips_per_unit = P × 6`; (4) `09 05 01 XX XX` TAB_SEQ handler records `XX × twips_per_unit` on the paragraph; (5) `0f 04` SI-TAB handler records B1 column position similarly; (6) RTF emits `\tx{twips}` per-paragraph; (7) DOCX writes `w:tabs`/`w:tab` XML. 10 new tests. 99/99 tests passing.

- [x] **MEMORIAL.002 / multi-file control-sequence artefacts** — Fixed in branch `feature/memorial-ctrl-sequence-fixes`. Three targeted fixes: (1) Pre-body `00 00` NUL handler now only jumps to the next `22 XX 0b` when content has been accumulated — avoids skipping text that follows the NUL pair in MEMORIAL-style documents (e.g. "Place: HEXHAM."); (2) Body `22 XX 0b e8 05` blocks flush the current paragraph (MEMORIAL-style paragraph break used instead of `13 04 50`); page-break variant `22 XX 0b e8 05 07 03 ...` additionally jumps past the binary layout block; (3) Self-referential ctrl sequences `22 XX XX` (type == ctrl_byte, e.g. `22 61 61`) are now identified as binary layout metadata and skipped to the next `22 XX 0b` rather than stopping at the first printable byte — eliminates `xd`-style artefacts in MEMORIAL.002 and BUILDNGS.H; (4) `22 XX 0b` paragraph blocks with SI tab embedded in header (B5=`0x0f` B6=`0x04`) are now skipped with a 9+3-byte skip — eliminates `a  A  HEXHAM` column-entry prefix artefacts. 4 new tests. 103/103 tests passing.

- [x] **`_find_body_start` incorrectly skips true body-start in `22 6d` files** — Implemented in branch `feature/fix-body-start-detection-22-6d`. Two fixes: (1) `_find_body_start` fallback scan now tracks `seen_high_b3`; once any B3≥0x80 block has been seen, the first low-B3 non-zero block is accepted as body start regardless of B5 — correctly identifies the `22 6d 0b 42 0d 14 78 00` body-start block in BUILDNGS.A-C and D-G. (2) `_skip_ctrl_sequence` gains a scan-forward case for `22 6d` (and other non-`22 61`) files: when a `78 00` block has no valid extended-variant doubled pair, scans past structural trailing bytes to the next `11` alignment marker or `22 XX` control prefix, preventing printable structural bytes leaking as text artefacts (`x`/`#`). The scan is restricted to non-`22 61` files so standard HENCOTES body blocks are unaffected. Confirmed: BUILDNGS.A-C `body[0]='Contents'` (was `'xContents'`), BUILDNGS.D-G `body[0]='Page'` (was `'# Page'`), HENCOTES golden fixture unchanged. 4 new tests. 117/117 tests passing.

- [x] **`1e 74` prefix byte variant** — Implemented in branch `feature/1e-prefix-variant`. Investigation documented in [`BINDINDX-Investigation.md`](BINDINDX-Investigation.md). Key changes: (1) `_detect_ctrl_byte` renamed to `_detect_variant(data) -> tuple[int, int]` — now scans both `0x22` and `0x1e` prefix bytes and returns `(prefix_byte, ctrl_byte)` for the most-frequent `XX YY 0b` pair; (2) `prefix_byte` threaded through `_find_content_start`, `_find_body_start`, `_find_footer_start`, `_skip_ctrl_sequence`, and `parse()`; (3) `0f 01`/`0f 02` paragraph separator handler extended to match `prefix_byte` (not hardcoded `0x22`); (4) doubled-pair threshold dropped to `0x00` for `1e` files (`min_dp = 0x00`); (5) doubled-prefix self-referential skip added (`1e 1e 74 …` → skip to next `para_ctrl`); (6) `11 06` alignment at B5/B6 of block header handled for `1e` variant; (7) literal-text fallback for `22 XX` guarded to `prefix_byte == 0x22`; (8) footer extracted from `22 6d` pre-body zone via recursive `parse()` call; (9) `22 6d 6d` self-referential sequences embedded in `1e 74` body blobs detected and skipped to prevent `"mmxd`-style artefacts. 8 new tests. 126/126 tests passing. Reference file: `Additional info/BINDINDX.HEX` (70,864 bytes, ~1,539 paragraphs extracted vs 1 before fix).

- [ ] **`02 06 02` pattern should emit ` - ` not `   `** — The `06` byte between two word separators (`02 06 02`) currently emits a space, giving triple-space output. Evidence across HENCOTES and BUILDNGS.H suggests this pattern is used as a dash separator (e.g. "Henhouse - domestic hens", "6.3.2015 - 100 Years Ago", "source - though not acknowledged by the OED - was cockfighting"). The fix is likely to emit `-` rather than ` ` when `06` is adjacent to word separators, but needs investigation to confirm it doesn't break other uses of `06` (e.g. genuine hyphenation between printable chars, or `06` adjacent to non-printable bytes).

- [x] **`22 XX 0b` block mid-sentence swallows formatting sequence** — Implemented in branch `feature/fix-mid-sentence-block-formatting`. When B5=`13` and B6=`04` in a `22 XX 0b` block, the skip is shortened to 5 bytes, leaving `13 04 XX` for the main loop. Fixes italic-on being consumed (OED now correctly italic in HENCOTES), spurious `\n` eliminated, and 2 previously-lost paragraph breaks now correctly fire. Confirmed safe across 447 occurrences in sample files — B7 is always `0x50`, `0x64`, `0x78`, or `0x8c`. Golden fixture regenerated (70 → 72 paragraphs). 3 new tests. 113/113 tests passing.

- [x] **Contents Page control block investigation** — Implemented in branch `feature/contents-page-control-block`. Investigation documented in `Contents Page Investigation.md`. Three fixes: (1) `78 00` extended-variant skip now also accepts `0x0a` as separator byte (excludes structural header blocks); (2) `0f 02 22 ctrl 0b` recognised as paragraph separator in non-`22 61` files — flushes paragraph; (3) `0f 01 22 ctrl 0b` recognised as line break in non-`22 61` files. Also added `_skip_ctrl_sequence` variants for B6=`0f` B7=`04` and B7=`0f` to leave SI sequences for the main loop. Fixes 105 affected `22 6d` sample files. 4 new tests. 110/110 tests passing.

- [x] **Section break creates spurious paragraph split in flowing text** — Implemented in branch `feature/section-break-paragraph-split`. Investigation documented in [`Section Break Investigation.md`](Section%20Break%20Investigation.md). The distinguishing signal is the byte immediately before `0e`: a word separator (`0x02`) or printable character means the sentence is in progress (layout switch only — no paragraph flush); a `13 04` sequence means the paragraph was already closed (real boundary — flush as before). The body-only guard (`i >= body_start`) ensures section breaks that delineate header/footer from body always flush correctly. Golden fixture regenerated (74 → 70 paragraphs, 6 spurious splits removed, 2 new genuine paragraphs from HENCOTES). 3 new tests. 106/106 tests passing.

- [ ] **JOY format parser** — 233 JOY files in the sample set cannot currently be converted.
