- [x] **Section break creates spurious paragraph split in flowing text** — Implemented in branch `feature/section-break-paragraph-split`. - [ ] **Footer alignment — tab-stop layout** — The page number zone (`11 06`, centre-aligned) and document reference zone (`10 07`, right-aligned) share a single `Paragraph` object, so `right` alignment overwrites `centre` and the page number loses its proper positioning. The correct fix is a tab-stop layout: a centre tab at the page midpoint and a right tab at the right margin, with a `\t` emitted between the two zones. Requires deriving the usable line width from the file header (page width minus margins — location unconfirmed). Once the tab-stop positions are known, the parser should emit `\t` at the `10 07` zone boundary and set the two tab stops on the footer paragraph; converters already handle `\tx` tab stops for RTF/DOCX.

- [ ] **Right alignment control sequence investigation** — `10 07` appears 5016 times across 393 sample files. The file encodings reference suggests `10 07` may consume additional parameter bytes. Investigation needed: identify what the `1d 66 XX` prefix represents, whether it is always present after `10 07`, and whether the current 2-byte skip leaves parameter bytes that should be consumed, potentially causing artefacts in right-aligned paragraphs.

- [ ] **Bibliography paragraph style codes** — The `08 05 01 XX XX` (PARA_INDENT) sequence uses XX ≥ `0x20` as a paragraph **style identifier** for bibliography/reference entries rather than a left-indent value.
